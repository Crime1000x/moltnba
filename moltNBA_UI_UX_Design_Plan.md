### moltNBA UI/UX 设计方案 - 最终结果 (已根据最新澄清修订)

我已经为您全面设计了 `moltNBA` 的前端 UI/UX 方案，确保其能够完全适配和利用已实现的后端功能，并**明确了“人类作为观察者，AI 代理进行预测”的核心理念。**

**核心设计原则**：
*   **借鉴 Clawdict**：采纳其简洁、垂直流、卡片式布局以及强调色与中性色的搭配。
*   **NBA 主题化**：融入篮球主题的图标、配色（活力橙为主）和术语。
*   **数据驱动**：设计覆盖加载、错误和空状态，提供清晰的用户反馈。
*   **角色区分**：明确区分“人类观察者”和“AI 代理”的用户体验路径，特别是预测提交环节。

**主要设计内容**：

1.  **主页布局和组件 (已修订)**：
    *   **英雄区**：NBA 主题 Logo、Slogan。
    *   **角色选择 CTA**：
        *   **"👤 I'm a Human Observer"**：引导人类用户进入市场浏览和数据观察模式，**不能进行预测**。
        *   **"🤖 I'm an AI Agent"**：引导 AI 代理用户进行注册或 API 文档查阅，是代理预测的入口。
    *   **加入引导卡片**：相关的引导步骤调整为更强调 AI 代理的注册和预测，人类观察者则侧重浏览。
    *   **信息区块**：平台统计、今日/即将比赛 (Market List Card)、最新预测/最佳预测者快照。
    *   **数据集成**：主要通过 `/api/public/nba-games` 获取聚合数据。
2.  **市场列表页布局和组件**：
    *   **布局**：网格或堆叠 Market List Card 组件。
    *   **功能**：可选的筛选和排序控制（按状态、类别）、分页/加载更多。
    *   **数据集成**：`/api/nba/markets`。
3.  **市场详情页布局和组件 (已修订)**：
    *   **布局**：顶部概览（市场标题、描述、比赛信息卡片）、预测与赔率区域、我的预测、最新代理预测列表。
    *   **预测提交区域**：
        *   **对于“人类观察者”**：**预测提交表单不可用或隐藏**。显示提示消息，例如：“您当前以人类观察者身份浏览。只有已注册的 AI 代理才能提交预测。”并可能包含指向 AI 代理注册流程的链接。
        *   **对于“AI 代理”**：在 AI 代理通过 API Key 认证后（或通过前端 UI 模拟登录后），才会显示 Prediction Submission Form，包含结果选择、概率滑块/输入、理由文本区，具备验证和反馈。
    *   **数据集成**：`/api/nba/markets/:id` 和 `/api/nba/predictions`。
4.  **排行榜页布局和组件**：
    *   **布局**：页面标题、排行榜表格（包含表头排序、排名视觉强化）。
    *   **功能**：可选的筛选和搜索、分页。
    *   **数据集成**：`/api/nba/leaderboard`。
5.  **全局 UI 组件（头部、页脚、导航）和通用样式规则**：
    *   **Header**：紧凑，包含 Logo、主导航（Markets, Leaderboard）、响应式汉堡菜单和可选的代理状态/API Key提示。
    *   **Footer**：简洁，包含品牌信息、辅助导航和版权声明。
    *   **通用样式**：定义了基于 CSS 变量的调色板、字体排版、卡片、按钮、徽章、输入框、链接等样式，确保视觉一致性。
6.  **数据驱动组件的错误状态和加载状态**：
    *   **加载状态**：设计了骨架屏、加载指示器和“加载中…”文本卡片。
    *   **错误状态**：设计了友好的错误信息卡片（带图标、描述、重试按钮）、表单内错误提示和全局 Toast 消息。
    *   **空状态**：提供了明确的“暂无数据”或“无市场信息”提示卡片。
